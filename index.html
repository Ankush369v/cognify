Cognify Intelligent Amplified

Project Statement: Cognify is a full-stack, AI-powered learning platform that transforms raw study materials, code, and datasets into structured, actionable content — eliminating the need to switch between multiple tools for learning and productivity.

Problem Statement

Modern students face unprecedented information overload and are forced to juggle multiple separate tools for studying, content generation, code analysis, and data visualization.

Creating flashcards manually, generating quizzes from PDFs, analyzing datasets, and reviewing code requires switching between multiple platforms — resulting in productivity loss, cognitive fatigue, and poor knowledge retention.

Existing Solutions & Their Gaps

Quizlet: Manual flashcard creation, no AI generation
ChatGPT: Generic responses, no persistent library
Tableau/Excel: Complex UX, no AI-driven insights
Gap: No unified platform combining AI generation + EDA + Study Tools

Our Solution

Multi-Model AI: OpenAI (GPT-4o) + Hugging Face (Llama, Qwen) routing
5 Unified Tools: MCQs, Flashcards, Summaries, Code Tools, Data Visualizer
Python EDA Engine: Automated statistical analysis with Pandas & SciPy
Stack: React 19 + Node.js 22 + Express 5.1 + MongoDB Atlas

System Pipeline

1. File Ingestion & Upload

Users upload study materials via multipart form. Multer validates and streams files to Cloudinary as raw (CSV/code) or auto (PDF) resource types to prevent data corruption.

Cloudinary

2. Feature Service Processing

The JWT-authenticated request reaches the relevant service module. For documents: pdf-parse extracts text. For datasets: the xlsx library parses rows, then a Python subprocess runs eda_engine.py (Pandas, NumPy, SciPy) for statistical computation.

Node.js 22
EDA Engine

3. Multi-Model AI Routing

A unified routing layer maps the user's selected model to the correct provider at runtime. GPT-prefixed models → OpenAI. All others (Llama, Qwen) → Hugging Face Router. Adding a new model requires only one config line.

GPT-4o GPT-4o-mini
Llama 3.1 8B Qwen 2.5 Qwen Coder

4. Persist & Deliver

The controller saves the parsed AI response as a GeneratedContent document in MongoDB Atlas (linked to the user). The result is returned to React, which renders the appropriate component — MCQ cards, flashcard deck, summary, code review, or interactive charts.
